When you are in the MONTAGE, you will be in your home directory (absolute path: `/ibm/gpfs/home/[user]` ). By default, users are allocated 500 Gb of storage space.

Your work will mostly revolve anything within your home directory. Hopefully you are not going to work with anything outside these folders; these would be within the admin jurisdiction, or somebody else's files and folders!

If you installed softwares via **anaconda** or **pip**, or download Docker/Singularity container images, they would be stored in your home directory.

:::info
If you worked on the Massive M3 HPC, the project structure is very different. You just need to think of your own directory. No scratch directories, no project directories. Simple.
:::

<blockquote>But Marcus, I need more storage space...</blockquote> 

Now, say if the resource that you are using can be shared, maybe your labmates are using the same resources, so it would be ideal if those files can be in a folder with shared-access. If your workflow requires a 50 Gb database, we dont need every single user to be having multiple copies of that 50 Gb dataset in each user folder, that takes too much space! So, we have another directory that is meant for shared resources (absolute path: `/ibm/gpfs/[department-abbreviation]` ).
