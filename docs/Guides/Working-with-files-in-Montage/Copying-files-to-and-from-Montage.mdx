There are several ways to transfer data between your local machine and Montage. We recommend the **Web File Browser** for most users as it requires no installation.

## 1. Web File Browser (MONTERA) - Recommended

For a no-install experience, you can use MONTERA's built-in web file browser. This is ideal for quick uploads/downloads and previewing files directly in your browser.

- **Access Link**: [http://10.168.151.1:3000/file-browser](http://10.168.151.1:3000/file-browser)

### How to use:
1. **Launch**: Navigate to the **File Browser** tab in MONTERA, select your session duration, and click **Start File Browser**.
2. **Open**: Once the status is `RUNNING`, click **Open Interface**.

3. **Manage**: You can now drag and drop files to upload, or right-click to download.

## 2. GUI interface like FileZilla or WinSCP

If you prefer a dedicated desktop application, FileZilla is a robust choice for managing large file transfers.

1. **Download**: Go to [https://filezilla-project.org/](https://filezilla-project.org/) to download the FileZilla Client.
2. **Connect**: Open FileZilla and use the **Quickconnect** bar at the top:
   - **Host**: `10.168.151.1`
   - **Username**: Your Montage username
   - **Password**: Your Montage password
   - **Port**: `22`
3. **Quickconnect**: Click **Quickconnect** to establish the SFTP connection.

![FileZilla Quickconnect](https://enterprisedt.com/products/completeftp/doc/guide/images/filezillaquick.png)

:::warning Unknown host key
The first time you connect, you will see a prompt about an "Unknown host key". This is normal for a first-time connection to a new server. You can safely check "Always trust this host" and click **OK**.
:::

## 3. Through a CLI tool like `rsync` or `scp` 

For advanced users or automated workflows, command-line tools like `rsync` offer the most control and efficiency.

```bash
# To send files from local (e.g.: laptop) to MONTAGE
rsync [source_path] [user]@[ip-addr]:[dest_path]
rsync text.txt u0001@10.168.151.1:/ibm/gpfs/u0001/Project1/

# To copy from MONTAGE back to my laptop
rsync [user]@[ip-addr]:[source_path] [dest_path]
rsync u0001@10.168.151.1:/ibm/gpfs/u0001/Project1/final_output.txt ./text.txt

# Useful options
#         -r: Recursive (for folders)
#         -a: Archive mode (preserves permissions/symlinks)
#         -v: Verbose
#         -h: Human readable
# --progress: Show transfer progress
```

If you already have a JupyterLab session running, you can simply use the built-in file explorer to drag and drop files.
