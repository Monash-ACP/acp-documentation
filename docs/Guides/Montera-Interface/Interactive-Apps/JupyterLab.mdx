JupyterLab is the preferred interactive development environment for data science and machine learning on Montage. MONTERA allows you to launch dedicated JupyterLab servers with full access to the cluster's GPUs and pre-configured software environments.

### Resource Selection
- **Partition**: Select `cpu` for general data processing or `gpu` for model training.
- **GPU Type**: Choose specific MIG slices (e.g., `1g.10gb`) to optimize resource usage.
- **Compute Limits**: Specify the number of Nodes, VCPUs, and Memory (GB) required for your analysis.

### Environment Management
MONTERA simplifies python environment management through the **Environment Type** dropdown:
- **Anaconda / Conda**: Select from global cluster installations (e.g., `/app/anaconda/2024.02`) or your own custom conda environments.
- **Python venv**: Use standard Python virtual environments located in your home or project directory.

:::important JupyterLab Package Requirement
If you are using a **custom** conda environment or python venv, it MUST have the `jupyterlab` package installed (e.g., `pip install jupyterlab` or `conda install jupyterlab`). Without this, the shell will fail to launch the web interface.
:::

## Accessing JupyterLab

Once your session status is `RUNNING` in the **My Jobs** tab:

1. Click the **Connect** button.
2. You will be prompted for a password or token.
3. Copy the **Access Token** provided in the MONTERA interface (displayed above the connect button) and paste it into the Jupyter login page.

:::tip Kernel Persistence
Your Jupyter notebooks and installed packages remain persistent across sessions. However, the Slurm job itself has a time limit. Ensure you save your work and checkpoint your models regularly.
:::

## Ending Your Session
When finished, shut down your kernels from within JupyterLab and use the **Terminate** button in the MONTERA dashboard to stop the Slurm job and free up the resources for other researchers.
