For developers who prefer a full-featured IDE experience, MONTERA offers a web-based version of VS Code that runs directly on compute nodes.

## Setting Up VS Code

1. Select **VS Code Server** from the left panel.
2. **Configure**: Pick your desired resource configuration (CPU/RAM/GPU).
3. **Launch**: Click **Start VS Code**.

### The Access Token Workflow
For added security, VS Code and RStudio sessions require a unique **Access Token** to log in once the session is Running.

1. Once the status is **RUNNING**, a **Connect** button and an **Access Token** will appear.
2. Click the **Copy** icon next to the Access Token.

    <img src={require('@site/static/img/Guides/Connecting-to-Montage/Connecting-to-Montage-via-MONTERA_AccessToken.png').default}
    alt="Copy Access Token"
    style={{ width: '80%', borderRadius: '12px', boxShadow: '0 4px 12px rgba(0,0,0,0.1)' }}
    />

3. Click **Connect**. A new tab will open asking for your authentication. Paste the token you just copied.

    <img src={require('@site/static/img/Guides/Connecting-to-Montage/Connecting-to-Montage-via-MONTERA_AccessToken2.png').default}
    alt="Paste Access Token"
    style={{ width: '80%', borderRadius: '12px', boxShadow: '0 4px 12px rgba(0,0,0,0.1)', marginTop: '1rem' }}
    />

4. You now have a full VS Code instance with access to all your Montage files and high-performance compute resources!
