To help you manage your resources and track your progress, we provide several custom helper commands on Montage. Use these as your first point of reference for cluster health and job status.

## Quick Overview: `show_cluster`
This command provides a real-time summary of the cluster's CPU and GPU resource availability. Use this to check if there is space for your next job.

## Job Status & VRAM: `show_job`
This is your most important command for monitoring active workloads. It shows your running/pending jobs and, crucially, tracks **GPU VRAM usage** in real-time.

> [!IMPORTANT]
> Always monitor your VRAM! If your job is using significantly less than your requested slice (e.g., using 5GB out of a 40GB slice), consider using a smaller MIG slice next time to free up resources for others.

**Sample Output:**
```text
╒══════╤══════════╤══════════╤══════════╤══════════╤══════════╤══════════╤══════════╤══════════╤══════════╤══════════╕
│ JobID│ Partition│ Name     │ User     │ State    │ Time     │ Nodes    │ CPU      │ Memory   │ GPU (VRAM)│ GPU (util)│
╞══════╪══════════╪══════════╪══════════╪══════════╪══════════╪══════════╪══════════╪══════════╪══════════╪══════════╡
│ 12345│ gpu      │ train_v1 │ marcus   │ RUNNING  │ 01:20:00 │ hpc2-gpu1│ 4        │ 16G      │ 32G / 40G │ 85%      │
╘══════╧══════════╧══════════╧══════════╧══════════╧══════════╧══════════╧══════════╧══════════╧══════════╧══════════╛

Job history is hidden by default. To view past jobs, run: show_job -d <days>
```

## Disk Quota: `chk_quota`
Monitor your home directory and shared project storage usage to avoid job failures due to "Disk full" errors.

**Sample Output:**
```text
Home directory usage
```
