## Slurm Partitions and QoS

Partitions group nodes together, while **QoS** (Quality of Service) flags define your priority and wall-time limits.

<div className="hpc-table">

| Partition Name          | Default QoS | Nodes      | Description                        |
| :---------------------- | :---------- | :--------- | :--------------------------------- |
| **`cpu`** (Default)      | `normal`    | Comp [1-6] | General-purpose CPU computing nodes |
| **`gpu`**               | `normal`    | GPU [1-8]  | GPU-accelerated computing nodes    |

</div>

### Available QoS Levels
Always use a QoS flag to set your job's duration and priority.

<div className="hpc-table">

| QoS Name    | Max Wall Time | Priority | Usage Example               |
| :---------- | :------------ | :------- | :-------------------------- |
| **`short`**  | 2 hours       | 200      | `--qos=short` (Quick tests) |
| **`normal`** | 7 days        | 1        | `--qos=normal` (Default)    |
| **`long`**   | 28 days       | 0        | `--qos=long`                |

</div>

## GPU Selection and GRES

Montage uses **MIG** (Multi-Instance GPU) technology on some nodes to allow multiple users to share a single physical A100. For high-performance work, dedicated A100/H100 nodes are also available.

### 1. Requesting a GPU Slice (MIG)
Available on nodes `mum-hpc2-gpu[1-4]`. Request these using specific **GRES** strings:

<div className="hpc-table">

| Slice Type | GRES String      | VRAM  | Best For                     |
| :--------- | :--------------- | :---- | :--------------------------- |
| **Small**  | `gpu:1g.10gb:1`  | 10 GB | Small models, JupyterLab     |
| **Medium** | `gpu:3g.40gb:1`  | 40 GB | General training, Batch jobs |
| **Large**  | `gpu:7g.80gb:1`  | 80 GB | Large memory models (1 GPU)  |

</div>

### 2. Requesting a Full GPU (Dedicated)
Available on `mum-hpc2-gpu[5-8]`. Use these for multi-GPU training or maximum NVLink performance.

<div className="hpc-table">

| Resource Type | GRES String     | Nodes      | VRAM (per GPU) |
| :------------ | :-------------- | :--------- | :------------- |
| **Full A100** | `--gres=gpu:a100:1` | gpu[5-6] | 80 GB          |
| **Full H100** | `--gres=gpu:h100:1` | gpu[7-8] | 80 GB          |

</div>

### Example Job Configuration
```bash
#!/bin/bash
#SBATCH --partition=gpu
#SBATCH --qos=short          # Use short for quick start
#SBATCH --gres=gpu:3g.40gb:1 # Request a 40GB MIG slice
```
